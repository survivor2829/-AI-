## 核心功能（必须实现 - P0）

### 功能1: 图标字体正常渲染（问题2）

**现象**：HTML 预览图里所有圆形图标区域显示空心方块 ❑，应该是功能图标（全地形作业/5G远程操控/超长续航等）。

**根因假设**：
- 模板用了图标字体（iconfont / Font Awesome / material-icons / etc.）
- 字体文件通过CDN或相对路径引用
- Playwright 浏览器在服务器端渲染时：
  - 字体URL访问失败（CDN被墙/域名不通）
  - 或字体文件路径不对
  - 或字体没打包进镜像

**修复要求**：
- 字体资源必须能在Playwright服务端环境稳定加载
- 不能依赖外网CDN（中国服务器必炸）
- 字体加载失败时要有fallback（即使也是方块至少不影响布局）

**验收标准**：
- [ ] 新上传批次生成的 preview.png 里，所有图标位置显示清晰可辨的图标（非方块）
- [ ] curl 生产 preview.png 下载下来用图片查看器打开，肉眼可见完整图标
- [ ] Playwright console 日志里无 `font loading failed` / `404` / `net::ERR_NAME_NOT_RESOLVED` 关于字体的报错

---

### 功能2: 主图正常显示（问题3）

**现象**：HTML预览图里"首屏"位置（大标题下方）应该展示产品主图，当前显示为浏览器默认的"图片加载失败"占位符（绿色小图标）。

**根因假设**：
- 紧急3 改了 UPLOAD_DIR，但模板里 `<img src="{{ product_image_url }}">` 的 URL 拼接可能未跟上
- 或产品主图实际落盘在正确位置，但模板里拼接的URL 前缀错了（比如多/少了 `static/`）
- 或 Playwright 渲染时 file:// 协议读不到这个路径

**修复要求**：
- 找到所有产品主图 `<img>` 标签的URL来源
- 确保URL在 Playwright 渲染上下文里能解析到真实文件
- 与紧急3 的 UPLOAD_DIR 改动保持一致

**验收标准**：
- [ ] 新批次 preview.png 首屏位置显示真实产品主图（肉眼可见是产品照片，不是占位符）
- [ ] 浏览器直接访问 `http://124.221.23.173:5000/uploads/batches/<batch_id>/<product_name>/product_cut.png` 能看到抠图后的产品
- [ ] preview.html 源码里的 `<img>` src 用浏览器能直接访问

---

### 功能3: AI精修图正常显示（问题6）

**现象**：batch detail页的AI精修列显示"图缺失"占位符，但有"↓AI版"下载按钮（说明实际文件已生成）。

**根因假设**：
- 跟问题3同源——前端 `<img>` src 拼接错
- 或 result JSON 的 ai_refined_url 路径未对应到新UPLOAD_DIR
- 或前端JS在 `result.ai_refined_path` 为空/错时的fallback逻辑问题

**修复要求**：
- AI精修图的URL在前端能正确拼接
- 用户能在AI精修列看到缩略图（不是"图缺失"占位符）
- 点击下载和点击预览都能工作

**验收标准**：
- [ ] AI精修列显示真实精修图缩略图（不是灰色"图缺失"占位符）
- [ ] 点击缩略图 lightbox 放大 → 显示完整精修图（不是404）
- [ ] 点击"↓AI版"下载按钮 → 下载到本地一张正常的JPG图（不是.customization或其他乱格式）
